# ADR-005 — Internationalization

**Status:** Approved

**Date:** 2026-07-06

## Context

Orion is intended for multilingual environments while maintaining a language-neutral internal architecture.

## Decision

Orion shall separate language concerns into five independent layers:

1. Internal system language (English)
2. Business data
3. User interface
4. Printed documents
5. System reference data

Business data is never translated automatically.

Printed documents are independent of the user's interface language.

## Rationale

This separation avoids coupling business information to user interface language while supporting multilingual deployments.

## Consequences

- Source code remains English-only.
- Database identifiers remain English-only.
- Customer-entered data is stored exactly as entered.
- UI localization remains independent of reporting.
